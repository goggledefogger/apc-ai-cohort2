# Session 1 — Danny's Demo Showcase

## Purpose
Show the SPECTRUM of what one person can build with AI — from 3-minute prototypes to production tools. Be honest about tradeoffs ("this took 3 minutes, this took 3 days — here's why").

## Format
Flip through open browser tabs of deployed versions, narrating as you go. Share the GitHub repo links with students afterward. Keep it moving — 1-2 minutes per project, not deep dives.

## Suggested Demo Flow (Pick 4-5 for Session 1)

**For maximum wow:**
1. **Fireflower** or **Orbits** (visual wow — audio visualizer or physics sim)
2. **Agent Viewer Town** (multi-agent orchestration — previews course content)
3. **Animals on Things** or **Pets Sorting Hat** (fun, quick, relatable)
4. **Is This Good to Buy** or **LLM Music** (practical tool — "you can't buy this off the shelf")

**Key talking points:**
- "Every one of these was built with AI assistance"
- "Some took an afternoon, some took a weekend — none took months"
- "By the end of this course, you'll build things like these"

## After Demos — Discussion Prompts
- "What could you build like this for your own work?"
- "Where are you currently stuck with tools?"
- "What would you want that you can't just buy?"
- "How long would any of these take traditionally?" (weeks/months)
